# 技术方案文档 — 面试突击鸭（interview-helper）

> 版本：v1.0 ｜ 日期：2026-06-10 ｜ 配套文档：`01-需求分析.md`
> 版本号基于 2026-01 知识基线 + 调研，**动工前须用 `npm view <pkg> version` 逐一复核**。

## 1. 架构总览

单仓库 Next.js 全栈应用（App Router），无独立后端服务。

```
浏览器
  │
  ▼
Next.js 16 (Vercel 部署)
  ├─ 页面层    App Router + RSC —— 服务端组件直查数据库（题库/题目页，SEO）
  ├─ 交互层    Server Actions —— 写操作（收藏、刷题记录、后台 CRUD）
  ├─ AI 层     Route Handler /api/interview/[id]/chat —— 流式 SSE
  │              └─→ DeepSeek API（OpenAI 兼容，Vercel AI SDK）
  └─ 数据层    Prisma 7 + @prisma/adapter-neon
                 └─→ Neon PostgreSQL（scale-to-zero）
```

## 2. 技术选型

| 层 | 选型 | 版本基线 | 选型理由 / 注意事项 |
|---|---|---|---|
| 框架 | Next.js | 16.x | Turbopack 已默认；Cache Components 若仍 beta 不启用；React 19.2+ |
| UI 库 | React | 19.2.x | Server Components / useActionState 稳定 |
| 样式 | Tailwind CSS | 4.1.x | CSS-first 配置（`@theme`），无需 tailwind.config.js |
| 组件 | shadcn/ui | CLI 3.x | 选 Tailwind v4 模板；**所有组件按暖橙设计 token 改造，禁止默认样式直出** |
| ORM | Prisma | 7.x | 无 Rust 引擎；driver adapter `@prisma/adapter-neon`；**不需要 Accelerate** |
| 数据库 | Neon PostgreSQL | - | Vercel Marketplace 一键集成；免费层 0.5GB；注意冷启动数百 ms |
| 认证 | Better Auth | 最新稳定 | NextAuth v5 永久 beta 且团队并入 Better Auth，新项目直接用后者；邮箱密码 + GitHub OAuth |
| AI | Vercel AI SDK + `@ai-sdk/deepseek` | ai@5（若 6 已稳定则评估后锁定） | v4/v5/v6 的 useChat API 互不兼容，**锁定主版本再开工**；测试用 mock provider 注入 |
| LLM | DeepSeek `deepseek-chat` | V3.2 | OpenAI 兼容（base_url: https://api.deepseek.com）；国内直连；输出约 $0.42/M token |
| 校验 | zod | 4.x | Server Action 入参 + AI 结构化输出双重用途 |
| Markdown | react-markdown + shiki | 最新 | 题解渲染 + 代码高亮 |
| 单元/组件测试 | Vitest + Testing Library | 最新 | - |
| E2E | Playwright | 最新 | 每里程碑黄金路径冒烟 |
| 包管理 | pnpm | 最新 | - |

**刻意不用：** Elasticsearch（PG pg_trgm 足够）、Redis（无高并发场景）、独立后端（Server Actions 足够）、状态管理库（RSC + URL state 足够，AI 对话用 useChat 内置状态）。

## 3. 数据模型（Prisma Schema 设计）

```
User ──┬── Favorite ────── Question
       ├── StudyRecord ─── Question
       └── InterviewSession ── InterviewMessage

QuestionBank ──── QuestionBankItem ──── Question   （多对多+题序）
```

### Question（题目，核心实体）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | cuid | - |
| title | String | 题目标题 |
| slug | String @unique | SEO URL：`/questions/hashmap-resize` |
| answerBrief | String | **30 秒速记版**：纯文本 2-3 句 |
| answerDetail | String | **详解版**：Markdown 图文 |
| followUps | Json | **追问链**：`[{question: string, hint: string}]`，2-4 个 |
| difficulty | Enum | EASY / MEDIUM / HARD |
| tags | String[] | 如 `["Java","集合","高频"]` |
| viewCount | Int | 浏览量 |

### 其他表

- **QuestionBank**：name、slug、description、icon（emoji）、category（Java/前端/Python/数据库/网络/操作系统/AI 大模型/热门）、sortOrder
- **QuestionBankItem**：bankId + questionId（@@unique）+ order
- **User / Session / Account / Verification**：Better Auth 标准表 + 自定义 `role`（USER/ADMIN）
- **Favorite**：userId + questionId（@@unique）
- **StudyRecord**：userId + questionId + date（按天去重，热力图数据源）
- **InterviewSession**：userId、bankId、difficulty、targetRounds、status（ACTIVE/FINISHED）、report（Json?，结构见 §5）
- **InterviewMessage**：sessionId、role（INTERVIEWER/CANDIDATE）、content、order

### 索引要点

- Question.title / tags：pg_trgm GIN 索引（搜索）
- StudyRecord：(userId, date) 复合索引（热力图聚合）
- slug 全部唯一索引

## 5. AI 模拟面试设计

### 5.1 流程状态机

```
准备页(选方向/难度/轮数) → 创建 Session(ACTIVE)
→ 循环 [ AI 提问 → 用户作答 → AI 决策(追问 depth≤2 | 下一题 | 收尾) ]
→ 结束(按钮或轮数耗尽) → generateObject 生成报告 → Session(FINISHED) → 报告页
```

### 5.2 面试官 Prompt 要点

- 角色设定为所选方向的资深面试官
- **从题库实际抽取 N 道题**（带 questionId 注入系统提示词上下文）——保证提问与题库联动，报告薄弱点能回链题目
- 行为约束：一次只问一题；先简短反馈再追问或下一题；追问最深 2 层；候选人说「不会」给提示降级
- 单 session 上限 30 条消息（token 防失控）

### 5.3 报告 zod schema

```ts
const reportSchema = z.object({
  overallScore: z.number().min(0).max(100),
  dimensions: z.object({          // 5 维雷达图
    knowledge: z.number(), depth: z.number(), expression: z.number(),
    logic: z.number(), adaptability: z.number(),
  }),
  strengths: z.array(z.string()),
  weaknesses: z.array(z.object({
    point: z.string(),
    questionId: z.string().optional(),  // 回链题目
    suggestion: z.string(),
  })),
  summary: z.string(),
});
```

### 5.4 工程要点

- 路由 `/api/interview/[id]/chat`：`streamText` 流式 + `onFinish` 落库
- 服务端校验：session 归属（userId）+ 状态 ACTIVE
- 失败降级：调用失败提示重试，已落库消息不丢；中断可恢复（session 保持 ACTIVE）
- `lib/ai.ts` 通过环境变量切换 DeepSeek / mock provider（测试零成本）

## 6. UI 设计规范（暖橙活力风）

原型：`docs/superpowers/mockups/visual-style.html` 方案 A。

### 6.1 设计 token（Tailwind v4 @theme）

```css
@theme {
  --color-primary: #FF7D00;        /* 主橙 */
  --color-primary-light: #FFB01F;  /* 鸭黄（IP/强调） */
  --color-cream: #FFF7ED;          /* 奶油底 */
  --color-border-warm: #F3E7D8;    /* 暖灰描边 */
  --color-ink: #1A1A1A;
  --radius-card: 14px;
}
```

### 6.2 规范要点

- Hero 渐变 `#FFF7ED → #FFEDD5 → #FED7AA`（135deg）；渐变只用于 Hero 和报告页，内容区白底
- 吉祥物简笔鸭（SVG 组件，多姿态）：Hero、空状态、404、AI 面试官头像——IP 贯穿
- 卡片：白底 + 1px #F3E7D8 描边 + hover 上浮 4px + 橙描边过渡
- 难度标签胶囊：EASY 绿 / MEDIUM 橙 / HARD 红
- 数字/代码用 JetBrains Mono；中文系统字体栈
- **反 AI 味红线**：禁用紫蓝渐变、禁用 shadcn 默认灰黑配色直出、禁用无意义大图标堆砌

### 6.3 关键交互

- 题目页三层答案：速记版默认展开；详解折叠；追问链卡片**逐个点击翻开**（先想后看）
- AI 对话：流式打字机效果，面试官消息带鸭子头像
- 热力图：GitHub 风格、橙色 5 级梯度

## 7. 测试与质量策略

## 8. 环境与配置

| 变量 | 用途 |

## 9. 里程碑实施顺序

| 里程碑 | 内容 | 关键产出 |
|---|---|---|
| M0 脚手架 | 项目初始化、设计 token、布局骨架、数据库连通、CI 脚本 | 可运行空壳 + 主题成型 |
