# 需求分析文档 — 面试突击鸭（interview-helper）

> 版本：v1.0 ｜ 日期：2026-06-10 ｜ 状态：已与产品负责人确认
> 本文档是项目的需求基线，后续 AI 会话应首先阅读本文档与 `02-技术方案.md` 理解项目上下文。

## 1. 项目背景与定位

### 1.1 一句话定位

面向编程求职者的**面试刷题平台**：题库刷题 + AI 模拟面试，参考 [面试鸭](https://www.mianshiya.com/) 的产品形态，做出自己的视觉风格与 AI 差异化。

### 1.2 项目性质

**个人学习 / 作品集项目**。不做支付、会员、商业化模块；追求功能完整、代码质量、UI 独特性，作为技术实践与简历展示。

### 1.3 需求调研结论（2026-06 联网调研）

- **需求合理性：成立。** 2025/2026 届毕业生均超 1100 万，「题库 + 八股 + 面经」消费习惯成熟，付费意愿已被面试鸭（永久会员 99.9→399 元）、力扣 Plus 等多个产品验证。
- **红海警示：** 单纯「汇总题库」无差异化。牛客（全链路+社区）、力扣（算法判题）、面试鸭（八股题库+面试官题解）、CodeTop（题频索引）已覆盖主要场景。
- **市场空白（本项目的差异化依据）：**
  1. 「题库 × AI 模拟面试」一体化闭环——现有产品中题库与 AI 面试是割裂的两类产品（本项目核心差异化，已纳入范围）
  2. 分层答案展示——解决面试鸭「单题千字信息过载」的用户反馈（已纳入范围）
  3. 遗忘曲线管理 SRS、AI 岗垂直题库、语音表达训练（未纳入 MVP，见 §5 不做清单）
- **可借鉴开源项目：** [liyupi/mianshiya-next](https://github.com/liyupi/mianshiya-next)（功能模型）、[Snailclimb/interview-guide](https://github.com/Snailclimb/interview-guide)（Spring AI 模拟面试实现思路）、CodeTop（按公司/频度组织的数据模型思路）。

## 2. 目标用户与核心场景

| 用户 | 场景 |
|---|---|
| 校招/社招编程求职者 | 按方向（Java/前端/数据库/AI…）系统刷八股，面试前突击 |
| 同上 | 刷完题后用 AI 模拟面试检验「能不能讲出来」，根据报告回流复习薄弱点 |
| 平台管理员（项目作者） | 通过管理后台维护题库与题目内容 |

核心用户旅程（北极星路径）：
**浏览题库 → 刷题（速记→详解→追问三层） → AI 模拟面试 → 评分报告暴露薄弱点 → 回链题目针对性复习**

## 3. 功能需求（按里程碑）

范围策略：**纵向切片 MVP**——每个里程碑都是可运行、可验收的完整版本。

### M1 题库核心（无需登录即可用）

- **首页**：Hero（搜索框+吉祥物）、分类导航、题库卡片网格（含题目数/浏览热度）
- **题库列表页**：按分类筛选
- **题库详情页**：题目列表，支持难度筛选
- **题目详情页**（核心页面）：
  - 分层答案：① 30 秒速记版（默认展开）② 详解版（Markdown 图文）③ 追问链（2-4 个面试官追问，逐个点击翻开）
  - 难度/标签展示，浏览量统计
- **搜索**：按题目标题/标签全文检索（PostgreSQL ILIKE + pg_trgm）
- **种子数据**：AI 预生成约 300 道题，覆盖 8 个分类（Java、前端、Python、MySQL、Redis、计算机网络、操作系统、AI 大模型）

### M2 用户体系

- 注册/登录：邮箱密码 + GitHub OAuth（Better Auth）
- 收藏题目 / 取消收藏
- 刷题记录：浏览题目自动记录，个人中心展示**日历热力图**（GitHub 风格、橙色梯度）
- 个人中心：收藏夹列表 + 热力图

### M3 AI 模拟面试（核心差异化）

- **准备页**：选面试方向（题库）+ 难度 + 轮数 → 开始
- **面试对话页**：AI 面试官从所选题库实际抽题提问（与题库联动），多轮对话，支持追问（最深 2 层），流式打字效果
- **评分报告页**：总分 + 5 维雷达图（知识广度/深度/表达/逻辑/应变）+ 优势 + 薄弱点（**回链到具体题目**）+ 总评
- 面试历史可回看、中断可恢复；需登录使用
- LLM：DeepSeek API（OpenAI 兼容），测试环境用 mock provider

### M4 管理后台

- 管理员角色（role=ADMIN）
- 题库 CRUD、题目 CRUD（含三层答案编辑）
- 普通用户访问 /admin 被拒

## 4. 非功能需求

| 维度 | 要求 |
|---|---|
| UI 风格 | **暖橙活力风**（已选定原型方案 A，见 `docs/superpowers/mockups/visual-style.html`）：主橙 #FF7D00、鸭黄 IP 贯穿、奶油渐变 Hero、大圆角卡片。明确要求**避免通用 AI 生成感**（不用千篇一律的紫蓝渐变/默认 shadcn 风） |
| SEO | 题目/题库页 SSR + 语义化 slug URL |
| 性能 | 题库/题目页服务端渲染直查数据库；AI 对话流式响应 |
| 安全 | API Key 仅存环境变量；面试会话校验归属；管理后台校验角色；入参 zod 校验 |
| 成本 | DeepSeek 按量（约 $0.42/M 输出 token，可忽略）；Neon/Vercel 免费层 |
| 质量门槛 | 每里程碑：lint + typecheck + 单测 + E2E 全绿后才进入人工验收 |

## 5. 明确不做（YAGNI 清单）

- 支付/会员/商业化
- 小程序、IDE 插件、移动端 App（仅响应式 Web）
- Elasticsearch（数据量小，PG 全文检索足够）
- SRS 间隔重复、错题本（调研有价值，留作二期）
- 语音面试、暗色模式、国际化
- 公司题库/题频维度（留作二期）
- 用户生成内容（评论/讨论区）

## 6. 验收标准（每里程碑）

| 里程碑 | 黄金路径 E2E | 人工验收 |
|---|---|---|
| M1 | 首页→题库→筛选→题目三层答案展开→搜索有结果 | 浏览器走查 + UI 对照暖橙原型 |
| M2 | 注册→登录→收藏→个人中心热力图点亮→登出 | 浏览器走查 |
| M3 | （mock）开始面试→3 轮对话→报告渲染+薄弱点回链 | 真实 DeepSeek 打一场完整面试 |
| M4 | admin 建题→前台可见；普通用户访问 /admin 被拒 | 浏览器走查 |
